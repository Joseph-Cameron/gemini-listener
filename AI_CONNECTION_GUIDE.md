# Lovense PoC — AI Connection Guide

This document contains everything another AI needs to send events to the Lovense PoC system running on this Windows machine.

---

## Connection Details

| Parameter | Value |
|-----------|-------|
| **Public Endpoint** | `https://respectively-grammar-incomplete-vault.trycloudflare.com/trigger-ai` |
| **Local Endpoint** | `https://respectively-grammar-incomplete-vault.trycloudflare.com/trigger-ai` |
| **Method** | `POST` |
| **Content-Type** | `application/json` |
| **Auth Token** (Socket.io only) | `change-me-in-production` |
| **Max Payload Size** | 10 MB |
| **Dashboard URL** | `https://respectively-grammar-incomplete-vault.trycloudflare.com/dashboard.html` (local only) |

**Remote AIs (Gemini, etc.) must use the public endpoint.** The HTTP endpoint (`/trigger-ai`) does NOT require a token — it is open. The auth token is only used for Socket.io WebSocket connections (the local listener and dashboard).

> **Note:** The public URL is generated by Cloudflare Tunnel and may change if the tunnel is restarted. Always verify the current URL before connecting.

## How to Send an Event

### Minimal Request (event only)

```json
POST https://respectively-grammar-incomplete-vault.trycloudflare.com/trigger-ai
Content-Type: application/json

{
  "event": "User received a $10 tip"
}
```

### Full Request (all fields)

```json
POST https://respectively-grammar-incomplete-vault.trycloudflare.com/trigger-ai
Content-Type: application/json

{
  "event": "User received a $10 tip",
  "message": "Great stream, keep it up!",
  "url": "https://example.com/fan/123",
  "image": "<base64-encoded-image-string>"
}
```

### Field Reference

| Field | Type | Required | Description |
|-------|------|----------|-------------|
| `event` | string | **Yes** | Natural-language event description. This is parsed by the AI function to determine device command and intensity. |
| `message` | string | No | Any text to display on the live dashboard and log on the local listener. |
| `url` | string | No | A clickable link displayed on the dashboard. |
| `image` | string | No | Base64-encoded image data (PNG, JPG, etc). Displayed on dashboard and saved to disk by the listener. |

## Response Format

```json
{
  "status": "dispatched",
  "parsed": {
    "command": "Vibrate",
    "intensity": 8,
    "reason": "$10 tip"
  },
  "hasMessage": true,
  "hasUrl": true,
  "hasImage": false
}
```

---

## Event Parsing Rules (Current Mock AI)

The `event` string is parsed by a rule-based function. These rules determine what device command and intensity are sent to the hardware.

| Input Pattern | Command | Intensity | Reason |
|---------------|---------|-----------|--------|
| Contains `$1` to `$5` + "tip" | Vibrate | 2 (low) | `$N tip` |
| Contains `$6` to `$15` + "tip" | Vibrate | 8 (medium) | `$N tip` |
| Contains `$16+` + "tip" | Vibrate | 15 (high) | `$N tip` |
| Contains "subscribe" or "follow" | Vibrate | 3 | `new subscriber` |
| Contains "gift" or "donation" | Vibrate | 10 | `gift received` |
| Anything else | Vibrate | 1 | `unrecognized event` |

Intensity range is 0–20 (0 = off, 20 = max). The mock parser currently caps at 15.

## Socket.io Direct Connection (Advanced)

If the other AI needs a persistent WebSocket connection instead of HTTP polling:

```javascript
const { io } = require("socket.io-client");

const socket = io("https://respectively-grammar-incomplete-vault.trycloudflare.com", {
  auth: { token: "change-me-in-production" }
});

// Listen for all device triggers in real time
socket.on("trigger_device", (payload) => {
  console.log("Event received:", payload);
  // payload = { command, intensity, message, url, image }
});
```

## Architecture Summary

```
  Other AI (you)
       │
       │  HTTP POST /trigger-ai
       ▼
  Web Dispatcher (Express + Socket.io, port 3000)
       │
       │  Socket.io "trigger_device" event
       ├──────────────────────────┐
       ▼                          ▼
  Local Listener              Dashboard
  (forwards to Lovense       (browser UI at
   hardware at                /dashboard.html)
   127.0.0.1:20010)
```

---

## Notes

- The public URL is provided by a Cloudflare Tunnel and may change if the tunnel restarts. The local dispatcher always runs on port 3000.
- Services auto-start on Windows login via a shortcut in the Startup folder.
- The dashboard token is hardcoded as `"dashboard"` — it's read-only and cannot trigger events.
- The `/trigger-ai` endpoint has no rate limiting currently.
